How to Create Articles

We welcome all forms of free expression. Below are some tips for creating articles that work well with our IPFS‑based platform:

  • HTML or Markdown: We recommend creating articles as HTML files or Markdown (which can be converted to HTML). This format allows for rich content, including animated GIFs and interactive elements.
  • Exporting from Medium: If you write on Medium, consider using the browser extension SingleFile. Export your article so that it retains its original formatting for IPFS upload.
  • Long Form Articles: For in-depth content, long-form HTML is preferred. Note that if you choose PDF, it will be static and won’t support animations.
  • Self-Contained Files: Aim to embed all images and styles within the file. Tools like Pandoc (using the --self-contained flag) can help.

Following these guidelines ensures your articles display correctly on our platform.

How to Submit Articles

We offer two flexible methods for submitting your articles while preserving your privacy:

  1. Submission Form: Use our secure submission form. Your article is encrypted in your browser and then transmitted over the decentralized Matrix network to our processing system. We then retrieve, decrypt, and upload your article to IPFS while indexing it into the blog. Since our system does not directly handle your connection, we cannot see your IP address. (Note: The Matrix network does use IP addresses to connect nodes, so if privacy is a concern, consider using a VPN or Tor.)
  2. Direct IPFS Upload: Alternatively, you can upload your article directly to IPFS using your preferred method or a hosted service (for example, Pinata or web3.storage). Once you obtain the file’s CID, submit the CID and required metadata using our dedicated submission form. If your article is encrypted (using our browser‑based encryption tool), we will decrypt it before publishing, ensuring no trace remains of you as the author.

Once approved, articles are pinned on our IPFS node and listed here.

Note: To help keep content online and censorship-resistant, feel free to pin any approved articles on your own IPFS node.

JSON Submission Template

Use the following JSON template when submitting your article via our IPFS submission form. Note that the title must not exceed 50 characters.

{
  "title": "Block title",
  "cid": "cid code",
  "author": "Blogger handle",
  "date_approved": "Submission date (YYYY-MM-DD)",
  "summary": "Summary of article for display",
  "tags": ["tag1", "tag2", "tag3"],
  "mature": false
}

Example:

{
  "title": "Why D&D Enthusiasts Make Epic Scrum Teams",
  "cid": "bafybeiadoue3nhvxlexlmxb5aelfaopcg4xqpt44fultumrgqctf323skq",
  "author": "TheNewAutonomy",
  "date_approved": "2025-03-11",
  "summary": "Running a scrum like an epic DnD Dungeon Master",
  "tags": ["scrum", "agile", "development", "project-management"],
  "mature": false
}

Custom URL & Branding

You can create a custom view of your articles by using URL parameters to automatically filter results and apply your custom branding. For example, appending the query string: 

?author=YourAuthorName&brand=YourBrandName

will:

  • Filter articles to show only those written by YourAuthorName.
  • Load an external CSS file named custom-YourBrandName.css for custom styling.

To have your custom CSS included, either submit your custom CSS file via a pull request on our GitHub repository or follow our contribution guidelines.
