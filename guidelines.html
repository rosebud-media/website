How to Create Articles

We welcome all forms of free expression. Below are some tips for creating articles that work well with our IPFS‑based platform:

  • HTML or Markdown: We recommend creating articles as HTML files or Markdown (which can be converted to HTML). This format allows for rich content, including animated GIFs and interactive elements.
  • Exporting from Medium: If you write on Medium, consider using the browser extension SingleFile. Select your article and export it so that it can then be imported to IPFS with its original formatting.
  • Long Form Articles: For in-depth content, long-form HTML is preferred. However, if you choose to create a PDF, note that it will be static and won’t support animations.
  • Self-Contained Files: Aim to have all your images and styles embedded in the file. Tools like Pandoc (using the --self-contained flag) can help with this conversion.

By following these guidelines, you help ensure that your articles display correctly.

How to Submit Articles

We like to give flexibility to bloggers while protecting your privacy. Below are some tips for submitting your articles to the platform.

  • Access IPFS: Install IPFS or create an account with a free IPFS host such as Pinata or web3.storage.
  • Upload file to IPFS: Using the local GUI if running locally or the dashboard if using a hosted service, select your file and import it. This will create a new CID code that identifies the file on the network.
  • Submit your CID: Email the completed post description using the template below to submissions@rosebud.world. Emails are deleted once the article is imported.

This process will change to a more automated system over the coming weeks to remove the need for email while preserving privacy.

By following these guidelines, you help ensure that your articles remain accessible on IPFS.

JSON Submission Template

Use the following JSON template when submitting your article. Note that the title must not be more than 50 characters.

{
  "title": "Block title",
  "cid": "cid code",
  "author": "Blogger handle",
  "date_approved": "Submission date (YYYY-MM-DD)",
  "summary": "Summary of article to show in list",
  "tags": ["tag1", "tag2", "tag3"],
  "mature": false
}

Example:

{
  "title": "Why D&D Enthusiasts Make Epic Scrum Teams",
  "cid": "bafybeiadoue3nhvxlexlmxb5aelfaopcg4xqpt44fultumrgqctf323skq",
  "author": "TheNewAutonomy",
  "date_approved": "2025-03-11",
  "summary": "Running a scrum like an epic DnD Dungeon Master",
  "tags": ["scrum", "agile", "development", "project-management"],
  "mature": false
}

Custom URL & Branding

You can create a custom view of your articles by using URL parameters to automatically filter results and apply your custom branding. For example, appending the following query string to the URL: 

?author=YourAuthorName&brand=YourBrandName

will:

  • Automatically filter the articles to show only those written by YourAuthorName.
  • Load an external CSS file named custom-YourBrandName.css to apply your custom styling.

To have your custom CSS included in the system:

  1. Email your custom CSS file to branding@rosebud.world with your desired brand name in the subject line.
  2. Alternatively, submit a pull request on our GitHub repository with your custom CSS file and follow the contribution guidelines.
For example, visit here to see a view filtering by the author thenewauthority specifying a custom css style.

How to Contribute

Rosebud World is a community project, and there are many ways you can contribute:

As a Content Creator

Share your insights and ideas by submitting your articles. We welcome diverse perspectives that enrich our community and promote free speech.

In Branding & Marketing

If you have expertise in design, branding, or marketing, help us enhance the look and outreach of Rosebud World. Contribute ideas, custom branding assets, and promotional strategies. Please reach out via email or contribute via a GitHub pull request.

As a Developer

Developers can help by improving the codebase, fixing bugs, and adding new features. Visit our GitHub repository to review open issues and submit pull requests. Your contributions help make Rosebud World more robust and user-friendly.
